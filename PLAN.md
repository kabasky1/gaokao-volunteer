# 高考志愿填报助手 — 开发计划

## 技术方案
- **后端**: Python + Flask
- **数据库**: SQLite
- **前端**: 单HTML页面（jinja2模板）
- **数据源**: gaokao.cn API（实时查+本地缓存）
- **启动方式**: 终端跑 `python3 app.py` → 浏览器打开 `http://localhost:5000`

## 项目结构

```
高考志愿填报助手/
├── app.py                 # Flask主服务
├── requirements.txt       # 依赖
├── data/
│   ├── schools.db         # SQLite数据库（学校库+录取数据缓存）
│   ├── import_schools.py  # 学校库导入脚本（一次性）
│   └── sync_scores.py     # 录取数据同步脚本（定时/手动更新）
├── static/
│   └── style.css          # 前端样式
└── templates/
    └── index.html         # 前端页面
```

## 数据流

```
用户输入(省份+分数+位次+选科)
      ↓
Python服务 → 查SQLite缓存 → 有数据? → 直接返回
                  ↓ 无数据
            gaokao.cn API → 查数据 → 写入SQLite → 返回结果
```

## 开发阶段

### P1：数据底座 — 学校库（半天）

**目标**: SQLite中有完整的学校列表+专业目录

**要做的事：**
1. 从教育部/阳光高考获取全国高校名单（约3000所），含名称+所在地+层次
2. 标记985/211/双一流标签（gaokao.cn API自带这些字段）
3. 导入SQLite，建表 `schools`
4. 建表 `provinces`（省份ID映射，gaokao.cn的省份编码）

**验证方式**: 查"广东省有多少所本科院校"能返回正确数字

### P2：录取数据缓存层（1天）

**目标**: 通过gaokao.cn API获取学校投档线和专业录取线，缓存到SQLite

**要做的事：**
1. 建表 `school_scores`（学校投档线：学校ID+省份+年份+科类 → 最低分+位次）
2. 建表 `major_scores`（专业录取线：学校ID+省份+年份+专业ID → 最低分+位次+选科要求）
3. 写API调用模块 `gaokao_api.py`：
   - `get_school_info(school_id)` → 学校基本信息
   - `get_school_score(school_id, year, province_id)` → 学校投档线
   - `get_major_scores(school_id, year, province_id)` → 专业录取线
4. 写缓存逻辑：查API前先查SQLite，有缓存直接用，没有则调API并写入
5. 批量预热脚本：遍历部分热门学校，提前缓存数据

**问题**: gaokao.cn的school_id未知。需要先获取学校ID列表。

**解决方案：**
- 从学校名称搜索API获取ID：`https://static-data.gaokao.cn/www/2.0/school/search.json?keyword=华南理工`
- 或从已有开源数据集提取学校ID对应关系

### P3：推荐算法（半天）

**目标**: 输入省份+分数+位次+选科，输出冲/稳/保推荐

**算法逻辑：**

```
用户位次 = 输入位次
学校投档位次 = 数据库中的最低位次

冲: 学校位次 × 0.95 ≤ 用户位次 ≤ 学校位次 × 1.05
    （位次接近，有机会冲进去）
稳: 学校位次 × 1.05 < 用户位次 ≤ 学校位次 × 1.20
    （位次明显高于投档线，比较稳）
保: 学校位次 × 1.20 < 用户位次
    （位次远高于投档线，基本稳上）
```

专业推荐类似，按专业录取位次匹配，同时检查选科要求是否满足。

**参数可调**，用户可以自己调冲/稳/保的阈值。
